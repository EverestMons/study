**Date:** 2026-03-10

## Overview

Add OCR capability for scanned/image-based PDFs. When `parsePdf()` detects a scanned PDF (>50% empty pages), instead of returning an error, render pages to images via pdfjs-dist canvas rendering and run OCR via tesseract.js v7.

---

## Dependency

**tesseract.js v7** — `npm install tesseract.js@7`

- Pure JavaScript/WASM OCR engine (Tesseract C++ compiled to WASM)
- Runs entirely client-side — no server, no sidecar
- Training data loaded from CDN on first use, cached by browser after download
- Tauri WebView compatibility: **LOW risk** — pdfjs-dist WASM already works in this app. Safari 16.4+ supports Web Workers + WASM + Relaxed SIMD. Include main-thread fallback if worker creation fails.
- CSP: `"csp": null` in tauri.conf.json — no restrictions, CDN fetch will work

## New File: `src/lib/ocrEngine.js`

### Exports

```js
export async function ocrPdfPages(doc, emptyPageNums, options)
```

**Parameters:**
- `doc` — pdfjs-dist document object (already loaded in parsePdf)
- `emptyPageNums` — array of 1-indexed page numbers that had <20 chars of text
- `options.onProgress(pageIndex, totalPages, pageNum)` — progress callback
- `options.languages` — array of language codes, default `['eng']`

**Returns:** `{ pages: [{ pageNum, text, confidence }], avgConfidence }`

### Implementation Blueprint

```js
import { createWorker } from 'tesseract.js';

const RENDER_SCALE = 2.0; // ~300 DPI equivalent for typical PDFs

export async function ocrPdfPages(doc, emptyPageNums, options = {}) {
  const { onProgress, languages = ['eng'] } = options;
  const langStr = languages.join('+'); // tesseract format: "eng+fra"

  // 1. Create single worker, reuse across all pages
  let worker;
  try {
    worker = await createWorker(langStr);
  } catch (e) {
    // Fallback: try without worker (main-thread mode)
    console.warn('[OCR] Worker creation failed, trying main thread:', e);
    worker = await createWorker(langStr, { workerBlobURL: false });
  }

  const results = [];

  try {
    for (let i = 0; i < emptyPageNums.length; i++) {
      const pageNum = emptyPageNums[i];
      if (onProgress) onProgress(i, emptyPageNums.length, pageNum);

      // 2. Render page to canvas
      const page = await doc.getPage(pageNum);
      const viewport = page.getViewport({ scale: RENDER_SCALE });
      const canvas = document.createElement('canvas');
      canvas.width = viewport.width;
      canvas.height = viewport.height;
      const ctx = canvas.getContext('2d');
      await page.render({ canvasContext: ctx, viewport }).promise;

      // 3. Run OCR on rendered page
      const { data } = await worker.recognize(canvas);

      results.push({
        pageNum,
        text: data.text || '',
        confidence: data.confidence || 0,
      });

      // Clean up canvas to free memory
      canvas.width = 0;
      canvas.height = 0;
    }
  } finally {
    // 4. Always terminate worker to free ~100-200MB RAM
    await worker.terminate();
  }

  const avgConfidence = results.length > 0
    ? results.reduce((sum, r) => sum + r.confidence, 0) / results.length
    : 0;

  return { pages: results, avgConfidence };
}
```

### Key Design Decisions

1. **Single worker, reused across all pages** — Creating a worker loads the WASM engine + training data (~15MB for English). Creating one per page would re-download/re-init each time. One worker handles all pages sequentially.

2. **`createWorker(langStr)` — v7 simplified API** — In tesseract.js v7, `createWorker('eng')` handles language loading + initialization in a single call. No separate `loadLanguage`/`initialize` steps needed.

3. **`tessdata_fast` training data (default)** — tesseract.js v7 defaults to `tessdata_fast` from CDN. This is the right choice: minimal accuracy difference vs `tessdata_best` for printed text, significantly faster.

4. **Only text output requested** — v7 defaults to text-only recognition. Don't enable `hocr`, `tsv`, or `blocks` output — saves 0.25-0.5s per page.

5. **Canvas cleanup** — Setting `width = 0; height = 0` after each page releases the pixel buffer. At 2x scale, a US Letter page is ~3,400 x 4,400 pixels = ~60MB per canvas. Without cleanup, a 50-page PDF would use 3GB.

6. **Worker termination in `finally`** — WASM memory (~100-200MB) must be freed. `finally` ensures cleanup even on error.

7. **Main-thread fallback** — If Web Worker creation fails in Tauri WebView, try `{ workerBlobURL: false }` which runs on the main thread. Slower but functional.

## Training Data Strategy

### CDN Loading (Default)
- Source: `https://tessdata.projectnaptha.com/` (tesseract.js default CDN)
- Downloaded on first OCR use per language
- Cached by browser/WebView after download
- English: ~15MB, most other languages: 2-15MB

### Language Configuration
- Default: English (`eng`)
- Stored in SQLite settings table: `getSetting('ocrLanguages')` → JSON array of language codes
- User can add languages via Settings modal
- Multiple languages loaded simultaneously: `createWorker('eng+fra+deu')`

## Integration Point in parsePdf()

### Current flow (error on scanned PDF):
```
extract text → detect >50% empty → return makeError()
```

### New flow (OCR on scanned PDF):
```
extract text → detect >50% empty → track empty page numbers
  → ocrPdfPages(doc, emptyPageNums) → merge OCR text with native text
  → continue normal heading detection + section building
```

### Changes to pdfParser.js (~15 lines modified)

1. **Track empty page numbers** during the page loop (line 85):
   ```js
   const emptyPageNums = [];
   // ... in loop:
   if (pageText.trim().length < 20) {
     emptyPages++;
     emptyPageNums.push(i);
   }
   ```

2. **Replace error return with OCR call** (lines 99-105):
   ```js
   if (numPages > 0 && emptyPages / numPages > 0.5) {
     const { ocrPdfPages } = await import('./ocrEngine.js');
     const ocrResult = await ocrPdfPages(doc, emptyPageNums, {
       onProgress: options?.onProgress,
       languages: options?.languages || ['eng'],
     });
     // Merge OCR text into pageTexts
     for (const ocrPage of ocrResult.pages) {
       const idx = pageTexts.findIndex(p => p.pageNum === ocrPage.pageNum);
       if (idx !== -1) {
         pageTexts[idx].text = ocrPage.text;
         pageTexts[idx].items = []; // No font-size items for OCR'd text
       }
     }
     // Mark as OCR'd for downstream fidelity tagging
     ocrUsed = true;
     ocrConfidence = ocrResult.avgConfidence;
     // Continue with normal section building...
   }
   ```

3. **Add `ocr` metadata to return value:**
   ```js
   return {
     ...existingReturn,
     _ocrUsed: ocrUsed || false,
     _ocrConfidence: ocrConfidence || null,
   };
   ```

4. **parsePdf signature update:** Add optional `options` parameter:
   ```js
   export async function parsePdf(buf, filename, options = {})
   ```

### Fidelity Tagging

The chunker (or parsers.js caller) checks `_ocrUsed` on the structured output:
- If `_ocrUsed === true`: set `fidelity: 'low'` on all chunks from this material
- If `_ocrUsed === false` or absent: keep default `fidelity: 'full'`

This is handled in `parsers.js` where the structured output is consumed, NOT in pdfParser.js itself.

## Performance Estimates

| Pages | Render Time | OCR Time | Total | Memory Peak |
|-------|------------|----------|-------|-------------|
| 1 | ~0.5s | ~2-4s | ~3-5s | ~250MB |
| 10 | ~3s | ~20-40s | ~25-45s | ~300MB |
| 50 | ~15s | ~100-200s | ~120-220s | ~350MB |
| 100 | ~30s | ~200-400s | ~240-440s | ~400MB |

- Render is fast (canvas GPU-accelerated)
- OCR is the bottleneck (~2-4s per page with `tessdata_fast`)
- Memory stable because canvas is cleaned per page; worker is single instance
- Worker WASM: ~100-200MB (one-time, freed on terminate)

## Risk Assessment

| Risk | Mitigation |
|------|-----------|
| tesseract.js WASM fails in Tauri WebView | Main-thread fallback via `{ workerBlobURL: false }` |
| CDN unreachable (offline/firewall) | Graceful error: "OCR requires internet for first use. Please connect and retry." |
| Very large PDF (100+ pages) | Progress UI + page-level processing prevents UI freeze. Consider page limit warning at 100+ pages. |
| OCR quality too low for meaningful extraction | Confidence threshold (Phase 3): if avgConfidence < 40, warn user. Content still stored with `fidelity: 'low'`. |
| Memory pressure on large documents | Canvas cleanup per page + single worker + worker termination in finally |
