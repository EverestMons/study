## STEP 1 — SA (Blueprint)

---

> You are the Study Systems Analyst. Read your specialist file at `study/agents/STUDY_SYSTEMS_ANALYST.md` first. Then read the diagnostic at `study/knowledge/research/material-skills-menu-diagnostic-2026-03-22.md` — it maps the full click path, data chain, and reusable components. Then read `src/StudyContext.jsx` (focus on `enterStudy` at lines 874-910 and `selectMode` at lines 913-1032) and `src/components/study/SkillPicker.jsx` (the existing picker). **Task:** Blueprint the material skills menu feature. When a user clicks "Start Studying" on a material card, the system shows skills extracted from that specific material so the user can pick what to study. **CEO decision:** If a material has exactly 1 skill, auto-select it but show a brief confirmation first (e.g., a small card: "[Material Name] → [Skill Name]" with a "Start" button, not the full picker). **Design these components:** (1) **DB query — `SubSkills.getByMaterial(materialId)`:** The exact SQL joining `chunks → chunk_skill_bindings → sub_skills`, filtered by `material_id`, with `unified_into IS NULL` and `is_archived = 0`. Return shape should match what `loadSkillsV2` produces (or specify what enrichment is needed after the query). (2) **`enterStudy` signature change:** Add optional `materialId` parameter. Specify how it flows through to `selectMode`. (3) **`selectMode("skills")` filtering:** After loading and enriching all skills, filter to only those returned by `getByMaterial` when a `materialId` is present. The diagnostic confirms SkillPicker renders whatever `pickerData.items` contains — no SkillPicker changes needed. (4) **Single-skill confirmation:** When the filtered list has exactly 1 skill, instead of setting `pickerData`, show a brief confirmation. Specify: is this a new state on StudyContext (e.g., `materialConfirm: { material, skill }`) that renders a small modal/card, or is it inline in the study screen? On confirm, call `bootWithFocus({ type: "skill", skill })`. On cancel, return to MaterialsScreen. (5) **Zero-skill edge case:** When a material has 0 skills (not yet extracted or extraction failed), show a message: "No skills extracted from this material yet." with a "Back" action. (6) **"Show all skills" escape hatch:** When viewing material-filtered skills in SkillPicker, add a way to expand to all course skills. Specify: is this a button in SkillPicker header, or a toggle? How does it interact with `pickerData`? **Also specify:** how `MaterialsScreen.jsx` changes (the onClick handler), what new state variables are needed (if any), and the data flow diagram from click to session start. **Blueprint must include** a "How to verify" section. **Deposit:** `study/knowledge/architecture/material-skills-menu-blueprint-2026-03-22.md`. Next is Step 2 (DEV). Stop here and wait for user confirmation before proceeding.
