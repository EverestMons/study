## STEP 2 — DEV (Implementation: add pre-questions + concreteness fading)

---

> You are the Study Developer. Read your specialist file at `study/agents/STUDY_DEVELOPER.md` first. Read the diagnostic at `study/knowledge/research/ies-prompt-diagnostic-2026-03-22.md` — it specifies exact insertion points for both changes. Read the IES spec at `study/docs/planning/ies-implementation-spec.md` sections 1.2 and 1.3 for the pedagogical rationale. **Task:** Add two teaching strategy instructions to the system prompt in `src/lib/study.js`. **Change 1 — Pre-questions (IES Rec 5a).** At the diagnostic's recommended insertion point, add a system prompt section that instructs the AI: when a student begins studying a skill (first message of a skill-focused or assignment-focused session), ask 1-2 brief diagnostic questions to activate prior knowledge before teaching. The questions should probe what the student already knows about the topic — not quiz them, but calibrate the teaching starting point. If the student clearly knows the material (answers demonstrate competence), skip ahead. If the student is blank, start from fundamentals. This should NOT fire on recap sessions (the student already knows they're reviewing) or explore sessions (unstructured). Only on skill-focused and assignment-focused sessions. **Change 2 — Concreteness fading (IES Rec 4).** At the diagnostic's recommended insertion point, add a system prompt section that instructs the AI: when teaching a new concept, start with a concrete, familiar example first (real-world analogy, specific numerical case, relatable scenario), then transition to the formal/abstract representation. Do not lead with definitions or formal notation — lead with "imagine you're..." or "think of a..." or a worked numerical example, then fade toward the general rule. This applies to all teaching sessions where new material is introduced. **Constraints:** Only modify `src/lib/study.js`. Do not change FSRS logic, context builders, or data loading. These are purely system prompt text additions. The instructions must coexist with the existing FACET-LEVEL ASSESSMENT and ASSESSMENT PROTOCOL sections without contradicting them. Keep the prompt additions concise — each should be 3-5 sentences max. Build verify: `cd study && npx vite build --mode development`. Commit with message: `"feat: IES pre-questions + concreteness fading in system prompt"`. Standard prompt feedback protocol → `study/knowledge/research/agent-prompt-feedback.md`. Next is Step 3 (QA).
