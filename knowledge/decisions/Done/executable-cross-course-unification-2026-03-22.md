## STEP 1 — SA (Blueprint: schema + merge engine)

---

> You are the Study Systems Analyst. Read your specialist file at `study/agents/STUDY_SYSTEMS_ANALYST.md` first. Then read the roadmap at `study/knowledge/decisions/roadmap-cross-course-unification-2026-03-22.md` — this contains CEO decisions and architectural questions. Then read `src-tauri/migrations/001_v2_schema.sql` (focus on `sub_skills`, `chunk_skill_bindings`, `concept_links` tables), `src-tauri/migrations/005_facets.sql` (facet tables), and `src/lib/conceptLinks.js`. **Task:** Produce a blueprint for cross-course skill unification — schema additions and merge engine design. **Schema change — Migration 008 `skill_courses` junction table:** `sub_skills.source_course_id` is already nullable (`ON DELETE SET NULL`). Add a `skill_courses` junction table: `skill_id INTEGER NOT NULL REFERENCES sub_skills(id) ON DELETE CASCADE, course_id TEXT NOT NULL REFERENCES courses(id) ON DELETE CASCADE, UNIQUE(skill_id, course_id)`. Include a JS backfill function that populates `skill_courses` from existing `sub_skills.source_course_id` values on first run (idempotent via settings flag). Also add a `unified_into INTEGER REFERENCES sub_skills(id)` column to `sub_skills` for soft-delete tracking — when a skill is absorbed, this points to the surviving skill. **Merge engine — `unification.js` module:** Design 2 functions: (1) `unifySkills(db, survivorId, absorbedId)` — atomic transaction that: re-points `chunk_skill_bindings` from absorbed to survivor, re-points `chunk_facet_bindings` where the facet belongs to the absorbed skill, merges facets (same concept_key → optimistic FSRS merge taking higher stability + lower difficulty; unique facets → transfer to survivor), re-points `assignment_question_skills` and `assignment_question_facets`, inherits concept links (dedup), creates `skill_courses` entry for survivor+absorbed's course, sets `absorbed.unified_into = survivorId`. (2) `detectAndUnify(db)` — scans concept_links for `same_concept` pairs where `sub_skill_a.source_course_id != sub_skill_b.source_course_id` and neither has `unified_into` set, determines survivor (more FSRS reviews via facet_mastery count), calls `unifySkills` for each pair. **Specify:** exact SQL for migration 008, function signatures with parameter types and return types, the optimistic facet FSRS merge formula, the concept_link dedup logic, how `unified_into` is used in queries (WHERE unified_into IS NULL filters out absorbed skills), and a "How to verify" section with acceptance criteria including a test scenario with 2 courses sharing one equivalent skill. **Deposit:** `study/knowledge/architecture/cross-course-unification-blueprint-2026-03-22.md`. Next is Step 2 (DEV). Stop here and wait for user confirmation before proceeding.
