# Practice Mode Spec

## Overview
Problem-set based practice for building skill mastery through progressive difficulty tiers. Problems are generated by AI, evaluated by AI, and scored to impact skill strength. Universal framework -- works for programming (code editor), math (equation input), and other domains (text input).

## Entry Flow
1. User selects skill from skill picker
2. Choice presented: **Learn** (dialogue) or **Practice** (problem set)
3. Current skill strength determines starting tier
4. AI generates 5 problems for that tier
5. Student works through problems, can skip and return within a tier
6. 4/5 pass required to advance to next tier

## Tier System

| Tier | Type | Strength Gate | Description | Base Points |
|------|------|--------------|-------------|-------------|
| 1 | Predict | 0-15% | "What does this output/evaluate to?" | 3 |
| 2 | Fill | 15-30% | Complete the missing piece | 5 |
| 3 | Write | 30-50% | Write a function/solution from scratch | 8 |
| 4 | Debug | 50-65% | Find and fix the error | 10 |
| 5 | Combine | 65-80% | Use multiple concepts together | 13 |
| 6 | Apply | 80%+ | Mini-program / complex problem | 16 |

Starting tier = highest tier the student's strength qualifies for. They can still advance through higher tiers in a single session.

## Scoring

```
Points per tier completion = base_points * attempt_multiplier

Attempt multiplier:
  1st pass (4/5 or 5/5): 1.0x
  2nd pass: 0.6x
  3rd pass: 0.35x
  4th+ pass: 0.2x

Journal rating:
  1st attempt pass -> "strong"
  2nd attempt pass -> "developing"
  3rd+ attempt pass -> "struggling"
```

Points feed into the existing skill strength/decay system via applySkillUpdates.

## No-Repeat Logic
- Every generated problem gets a short signature (1-line summary) stored in problemSignatures[]
- Generation prompt includes all previous signatures as "ALREADY USED"
- AI instructed to generate completely different scenarios, variable names, problem structures
- Signatures persist across sessions for the same skill

## Skip Logic
- Student can skip any problem within a tier
- Skipped problems remain accessible (not marked pass/fail)
- Cannot advance to next tier without 4/5 passed in current tier
- Skipped = unanswered, not failed

## Language Detection
- Inferred from skill name/description, not configured separately
- "Java for-loops" -> Java
- "Recursion" (no language qualifier) -> general pseudocode or course default
- AI generation prompt includes detected language for syntax-appropriate problems

## Data Model

```
PracticeSet {
  id: string,
  skillId: string,
  courseId: string,
  detectedLanguage: string | null,  // "java", "python", null for general
  currentTier: 1-6,
  tiers: {
    [tierNum]: {
      attempts: [{
        problems: [{
          id: string,
          prompt: string,
          starterCode: string | null,
          expectedApproach: string,  // for AI evaluator, not shown to student
          studentAnswer: string | null,
          evaluation: string | null,  // AI feedback text
          passed: bool | null,  // null = unanswered/skipped
        }],
        passCount: number,
        attemptNumber: number,
        completed: bool  // true when 4/5 passed
      }],
      passed: bool,
      pointsAwarded: number
    }
  },
  problemSignatures: string[],  // 1-line summaries of all generated problems
  startedAt: ISO string,
  lastActiveAt: ISO string
}
```

## AI Prompts

### Generation Prompt
```
Generate 5 practice problems for the skill: {skill.name}
Description: {skill.description}
Course: {courseName}
Language: {detectedLanguage or "use pseudocode"}
Tier {N}: {tier type description}

{tier-specific instructions -- see below}

Source material for reference:
{relevant material chunks}

ALREADY USED PROBLEMS (generate COMPLETELY DIFFERENT scenarios, variable names, and structures):
{problemSignatures joined by newline, or "None yet"}

Return ONLY a JSON array:
[{
  "id": "p1",
  "prompt": "the problem statement shown to the student",
  "starterCode": "code template if applicable, or null",
  "expectedApproach": "what a correct answer looks like -- for evaluation only, never shown",
  "signature": "one-line summary for dedup tracking"
}]
```

### Tier-Specific Instructions
- **Tier 1 (Predict):** Show a code snippet or expression. Ask what it outputs/evaluates to. No starter code. Student answers with the expected output.
- **Tier 2 (Fill):** Provide code with a clearly marked blank (___). Student fills in the missing part. starterCode has the template.
- **Tier 3 (Write):** Describe what a function/solution should do. Student writes it from scratch. No starter code.
- **Tier 4 (Debug):** Provide code with a bug. Student must identify and fix it. starterCode has the buggy code.
- **Tier 5 (Combine):** Problem requires this skill PLUS a prerequisite skill. Describe the task. May or may not have starter code.
- **Tier 6 (Apply):** Multi-step problem or mini-program. Real-world-ish scenario. Student builds a small complete solution.

### Evaluation Prompt
```
Evaluate this student's answer.

Skill: {skill.name}
Problem: {prompt}
{starterCode ? "Starter code: " + starterCode : ""}
Expected approach: {expectedApproach}

Student's answer:
{studentAnswer}

Evaluate on conceptual correctness and proper application of {skill.name}.
For code: minor syntax issues (missing semicolon, slight formatting) are OK if the logic is sound.
For Tier 1 (predict): answer must match expected output exactly or be semantically equivalent.
For Tier 2 (fill): the filled portion must make the code work correctly.

Return ONLY JSON:
{
  "passed": true/false,
  "feedback": "brief explanation of what was right or wrong, 2-3 sentences max"
}
```

## UI Layout

### Editor Behavior
- Monospace font, dark background
- Tab key inserts spaces (not focus change)
- No autocomplete, no syntax highlighting (keep it honest)
- Generous height (~12-15 lines visible, scrollable)
- If starterCode provided, pre-populated and editable

### Problem Navigation
- Dot indicators at bottom show problem status
- Can click dots to navigate between problems within tier
- Skip button advances to next unanswered problem
- After all 5 attempted/skipped, show tier summary

### Tier Completion Screen
- Shows 5 problems with pass/fail status
- Points awarded displayed
- Button: "Continue" (next tier or retry)

## Persistence
- PracticeSet saved to DB keyed by `practice-{courseId}-{skillId}`
- Auto-saves after each problem evaluation
- Resumable -- student returns to exact problem they left off
- problemSignatures persist across sessions to prevent repeats

## Integration with Existing Systems
- Skill points from practice feed into applySkillUpdates (same as chat)
- Session journal gets an entry when practice session ends
- Notifications (not chat) for tier advancement, points awarded
- Practice mode accessible from: skill picker AND assignment skill gaps
